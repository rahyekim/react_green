비동기 프로그래밍 Async

Asychronus code

        비동기 코드를 사용하면 프로그램이 (파일에서 데이터를 가져오는것과같은)
        시간이 오래 걸리는 작업을 시작하고
        첫 번째 작업이 완료되기 전에 다른 작업을 계속 진행할 수 있습니다.
        애플리케이션이 멈추는 것을 방지하는데, 이는 사용자 경험(UX)에 매우 중요

        일부 작업 완료하는데 시간이 걸림(네트워크 요청, 타이머, 사용자입력)
        반응성을 유지하기 위해 javascript는 비동기 프로그래밍을 사용
        비동기 흐름이란 자바스크립트에서 특정작업이 백그라운드에서 실행되고,
        결과가 준비되면 처리 될 수있도록 하는 방식.

        멈춘 페이지는 깨진 페이지(하얘....)
        비동기 코드는 실행을 차단하지 않음

        자바스크립트는 서로 다른 핵심 개념을 사용하여 비동기 프로그래밍을 처리

        🔹 Synchronus : 표준 흐름이 한 줄씩 실행
        🔹 Timers : 다른 코드가 대기하는 동안 코드가 실행될 수 있도록 함 
        🔹 Callbacks : 비동기 자바스크립트를 위한 최초의 해결책 > 콜백지옥
        🔹 Events : 실행 대기 중인 콜백 함수를 저장
        🔹 Promises 비동기 작업을 깔끔하게 처리하는 도구
        🔹 Async-await: 비동기 코드를 처리하는 깔끔하고 현대적인 방법

        🔸병렬 처리: 서로다른 process프로세스에서 여러 작업을 동시에 실행[더빠르게완료] 
                    멀티프로세스 -> 한번에 10000개의 이미지를 처리 
        🔸비동기 처리: 동시에 실행하는 것이 아니라 작업간 전환 []
                    프로세스 1개 -> 사용자가 스크롤하는 동안 코드실행
                    지금 이작업을 시작하세요 나는 결과를 즉시 알필요는 없어요
                    완료되면 나중에 알려주세용

                    3. 서버가 음식을 가져와요(콜백)
